## 2. 下一步：自研前端网页架构 (DIY Web UI Strategy)
**C++ 黑魔法保护 (Raw String Literal)**：
即使是自己写的 HTML/JS，里面也会充满双引号。坚决使用 `R"rawliteral(...)rawliteral"` 语法，把纯 HTML 无痛包裹进 C++ 中。

**前端 JS 控制逻辑设计**：
为了防止车撞墙，必须采用**“按住才走，松开就停” (Hold-to-drive)** 的逻辑。
- 绑定鼠标事件 (兼容电脑)：`onmousedown` 发送移动指令，`onmouseup` 发送停止指令。
- 绑定触摸事件 (兼容手机)：`ontouchstart` 发送移动，`ontouchend` 发送停止。

## 3. 后端 API 路由与安全机制 (Backend Routing & Safety)
前后端接口自己定，我说了算！
- 前端按“前进” -> JS 发送 `GET /forward` -> C++ 触发 `server.on("/forward", moveForward);`
- 前端按“左转” -> JS 发送 `GET /left` -> C++ 触发 `server.on("/left", turnLeft);`

**🚨 终极安全机制 (Watchdog / 防撞墙狗)**：
哪怕写了松手停，万一手指没松开的时候 **WiFi 突然断了**怎么办？车会一直保留最后一次的“前进”指令，直接冲出桌子或者撞毁！
**应对方案**：在 C++ 的 `loop()` 里加一个超时计时器 (Timeout)。每次收到任何方向指令，就重置倒计时。如果超过 500ms 没收到任何有效的心跳/移动指令，强制触发 `stopBoth()` 刹车！
